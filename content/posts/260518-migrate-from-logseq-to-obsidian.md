아이디어가 생각이 나거나 메모할 것이 있으면 그냥 아무데나 적어두는 편을 선호하는 사람으로, [LogSeq](https://logseq.com/)로 노트를 작성해오다가 최근에 다시 [Obsidian](https://obsidian.md/)으로 옮기게 되었다.

LogSeq는 그날의 Daily Journal이 기본 페이지로 열려서 바로 생각을 적어 넣으면 되는 반면, Obsidian에서는 내용을 적어 넣을 페이지를 먼저 만들거나 찾아야 하는 점이 내게는 걸림돌로 작용했다. Obsidian에서도 Daily Jounal을 만들 수 있으나 LogSeq 처럼 Daily Journal이 중심이 되는 것이 아니어서 자유로운 생각의 흐름에 방해가 되었다. Obsidian의 다양한 플러그인들과 옵션들이 도움이 되기 보다는 내게는 그것들을 설정하고 관리하는데 (공들여 설정한 Data View를 나중에 수정하려 하면 이전에 어떻게 설정했는지를 기억해 내는데 시간이 걸린다 던지) 많은 시간을 쓰게 된 점도 LogSeq를 사용하게 된 이유 였다.

그렇게 LogSeq를 사용해 오다 며칠 전 블로그 포스팅 작성을 위한 md 파일 에디터로 Obsidian을 다시 사용하게 되면서 Obsidian의 설정을 최소한으로 유지하면 LogSeq 보다 사용성이 좋다는 점을 새삼 깨닳게 되었다. Daily Note Editor 플러그인을 사용하면 Obsidian을 LogSeq의 Journal과 동일하게 사용할 수 있고, Obsidian의 Tags 플러그인은 LogSeq의 Tag플러그인이 `#`으로 시작하는 모든 문자열을 태그로 인식하는 거슬리는 문제가 없으며, 무엇 보다도 iPhone의 Obsidian App의 완성도가 높아서 iPhone에서 문서를 작성하기가 편하다는 점 등이 이유가 되어 마침내 다시 Obsidian으로 옮기게 되었다.

현재 사용 중인 최소한의 플러그인들과 설정은 다음과 같다.

- Minimal Theme
- Minimal Theme Settings
- Mononote
- Auto Link Title
- Calendar
- Mononote
- Update modified date
- Checklist
- Tags view

모두 7개의 플러그인과 1개의 Theme 그리고 한 개의 설정인데, 앞으로도 플러그인 10개를 넘지 않게 유지할 생각이다.

- 플러그인은 10개 이하로 유지한다.
- 불필요한 Pedefined Header나 Data View는 사용하지 않는다
 
Keep It Simple, Stupid.
