## 개요

Counselnote는 상담사와 내담자를 위한 세션 기록 및 일정 관리 웹 애플리케이션이다. 
개인 프로젝트로 기획, 데이터베이스 설계, 프론트엔드 및 백엔드 개발의 전 과정을 단독으로 진행했다.

## 제작 배경 및 목표

상담 진행 후 발생하는 기록 정리의 번거로움을 줄이고, 데이터를 효율적으로 관리하고자 기획했다.
초기에는 상담 오디오를 텍스트로 변환(STT)하고 핵심 내용을 자동 요약하는 기능을 핵심 목표로 삼았다. 그러나 오디오 파일 데이터 저장과 API 변환 과정에서 발생하는 비용적인 문제로 인해 해당 기능 및 구상은 최종적으로 제외되었다. (구체적인 비용 문제와 판단 기준에 대해서는 하단에 별도 작성 예정)

결과적으로 현재 기능은 상담사가 기록 자체를 효율적으로 남기고 관리할 수 있는 도구를 제공하는 것에 집중되었다.

## 기술 스택

- **Frontend/Backend**: Next.js (App Router)
- **Database/ORM**: PostgreSQL, Prisma
- **Authentication**: NextAuth.js
- **기타**: 파일 업로드 및 클라우드 연동 (초기 도입 예정이었던 AI/STT 모델은 제외)

## 주요 기능 (Features)

### 1. 대시보드 및 일정 관리
등록된 내담자 목록과 기본 인적 사항, 진행 중인 상담 상태를 표 형태로 제공한다. 
캘린더 모달 UI를 통해 전체 상담 일정을 직관적으로 파악하고 생성할 수 있으며, 구글 캘린더 연동을 위한 데이터 필드를 지원한다.

![대시보드 및 캘린더 화면 - 스크린샷 추가 요망](/images/counselnote/dashboard-calendar.png)

### 2. 내담자 및 세션 관리
내담자별 상세 페이지를 통해 인적사항과 과거 상담 기록을 한곳에서 관리한다. 
각 상담 세션은 '예정됨(Scheduled)', '완료됨(Completed)', '취소됨(Cancelled)' 상태로 트래킹된다.

![내담자 관리 및 세션 목록 화면 - 스크린샷 추가 요망](/images/counselnote/client-management.png)

### 3. 상담 기록 및 노트 작성
개별 세션마다 텍스트 형태의 노트(Note)를 자유롭게 작성할 수 있다. 
기존의 파일 형태 상담 일지 또한 업로드하여 관리하며, 작성한 코멘트, 별점(Rating) 기반 상담 평가, 부가적인 분석 링크(Analysis Link) 첨부 기능 등을 지원한다.

![상담 세션 노트 작성 및 기록 관리 화면 - 스크린샷 추가 요망](/images/counselnote/session-note.png)

## 개발 시 주요 고려사항 및 이슈

### 녹음 파일 저장 및 변환 기능 보류 (비용 이슈)
초기에는 외부 STT Provider를 통한 오디오 텍스트 변환 및 분석을 기획하고 비동기 처리 상태(`processing`, `completed` 등)까지 데이터베이스에 구현했다. 하지만 실제 테스트 및 적용 과정에서 스토리지 보관 비용과 서드파티 API 통신 비용 문제가 예상되어 부득이하게 제외되었다.

> [비용 문제 관련 구체적 설명 추가 요망]

### 데이터 정합성 보장
단일 상담 세션(`CounselSession`) 하위에 개별 노트(`Note`)와 기록(`CounselRecord`) 등 관련된 데이터 모델을 1:N으로 구성하여 데이터 무결성을 유지하고 추가 확장성을 고려했다.

## 배포 및 호스팅 환경

개인 사이드 프로젝트인 만큼 운영 유지비용을 최소화하기 위해 **전면 무료 호스팅(Free Tier) 서비스**를 조합하여 배포 환경을 구축했다.

- **Frontend/Backend 호스팅 (Vercel)**: Next.js 프레임워크를 개발한 Vercel을 배포 플랫폼으로 선택했다. GitHub Repository와 연동하여 푸시(Push)할 때마다 자동으로 빌드되고 배포되는 CI/CD 파이프라인을 추가 비용 없이 구축할 수 있으며, Next.js의 App Router 및 Server Actions 지원이 가장 완벽하고 안정적이기 때문이다.
- **Database 호스팅 (Supabase/Neon 등)**: PostgreSQL 데이터베이스 리소스에는 클라우드 DB의 무료 티어를 활용했다. (※ *Supabase 또는 Neon 등 현재 사용 중인 서비스로 최종 수정 요망*) 트래픽이 많지 않은 개인 프로젝트 환경에서 매월 발생하는 클라우드 인스턴스(AWS RDS 등) 고정 지출을 막으면서도 안정적인 RDBMS 환경을 유지하기 위한 타협점이었다.

## 마무리

Counselnote 프로젝트를 통해 기획부터 데이터베이스 설계, 그리고 현실적인 인프라 비용 문제로 인한 기능 타협까지 풀스택 서비스 개발 사이클을 전반적으로 경험할 수 있었다. 
작성한 초안 버전을 바탕으로 향후 데이터 시각화 컴포넌트 추가 및 모바일 반응형 페이지 작업을 진행할 계획이다.
